# 1. Introduction

A personal portfolio website serves as a professional platform to showcase a developer's technical skills, projects, education, and contact information. Unlike a traditional resume, it allows recruiters to explore real projects and evaluate development skills through an interactive interface.
The portfolio follows a modern dark-themed design with responsive layouts, smooth animations, interactive navigation, animated skill bars, project showcase cards, and client-side contact form validation.

# 2. Objectives

- Design a modern and professional portfolio website.
- Build a fully responsive layout for desktop, tablet, and mobile devices.
- Showcase technical skills using animated progress bars.
- Display projects with descriptions and technologies used.
- Provide recruiters with downloadable resume and contact information.
- Implement JavaScript animations and interactivity.
- Follow semantic HTML and maintainable project structure.

# 3. Scope & Limitations

This project is a frontend-only application.

Current limitations include:

- No backend server.
- No database integration.
- Contact form performs only client-side validation.
- Messages are not stored or emailed.
- Projects are static and manually added.

These limitations leave room for future backend integration using Node.js and MongoDB.

# 5. Signature Features

The portfolio includes the following signature features:

###  Typewriter Hero Introduction

A JavaScript typing animation displays multiple professional titles one after another in the Hero section.

###  Scroll-triggered Skill Bars

Skill progress bars animate from **0%** to their respective values when the Skills section enters the viewport.

### Hidden easter egg

A secret interactive feature has been added to the portfolio. When the user performs a predefined hidden action (such as Z  repeated logo clicks), a hi/dden achievement message is revealed, adding a fun and engaging element to the user experience.
